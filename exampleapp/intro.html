AOC intends to offer developers a more natural way of using javascript widgets and controls in the context of AngularJS applications. We all now that using plugins such as jQuery UI with angularJS applications can be awkward so this library tries to alleviate these problems by abstracting these plugins under a more streamlined interface

Setup

To use AOC you just have to reference the library and include the module 'aoc' in your application. code

Declaration

Once the previous step is done, all you need to do is add the 'aoc' directive and declare what plugins you would wish to use.

Simple jQuery-UI Tabs example: <div aoc="tabs tooltip" title="My tabs"> <ul> <li><a href="#e1tab1">Tab 1</a></li> <li><a href="#e1tab2">Tab 2</a></li> <li><a href="#e1tab3">Tab 3</a></li> </ul> <div> <div id="e1tab1">This is tab 1</div> <div id="e1tab2">This is tab 2</div> <div id="e1tab3">This is tab 3</div> </div> </div>

Configuration

Using configuration function

<script> var aoc = angular.module('aoc'); aoc.configure('dialog', function (scope, other, injectables) { return { actions: { default: ['open', 'close'], // default actions that are supported by the widget. Must be invokable through the defaultInvoker. custom: { // custom actions that can be user defined. editPerson: function (person) { scope.person = person; this.open(); //<- default action. This is resolved as the Control itself. }, defaultInvoker: function (element, controlName, args) { // Invoker of the default actions. By default it uses the //jQuery-ui widget style of accessing methods that is: $(element)[widgetName]('methodName',arguments); element[controlName].apply(element, args); } }, events: { // Events that will be attached to the affected element. 'dialogOpen': function () { scope.dialogOpen = true; } }, options: { autoOpen: true, title: '{{title}}', close: function () { scope.dialogOpen = false; }, dialogClass: 'edit-dialog' }, createFn: function (element, controlName, options) { // Instantiating function of the widget. By default it uses the //jQuery-ui widget style of instantiating widgets that is: $(element)[widgetName](options); return element[controlName](options); } } } }, '#editDialog'); /* the configuration will only be applied to elements matching this selector */ </script>

Using events and options directives

<div aoc="dialog" events="e" <- events attribute key options="{dialog:'d'}" <- options attributes keys, select one or each control used. d-auto-open="=true" <- the "=" sign specifies that the value of the attribute should be evaluated. d-title="@title" <- the "@" sign specifies that the value of the attribute is a angular binding and should be enclosed in {{ }}. d-close="+dialogIsOpen=false" <- the "+" sign specifies that the value should be evaluated as a function. d-dialog-class="edit-dialog" <- no sign specifies that the value should be evaluated as a string. e-dialog-open="dialogIsOpen=true" <- every event is evaluated as a function so there is no need to place a "+" sign. > ... </div>

Accessing the controls

Accessing the controls can be made through the $aoc helper function in the scope.

JS

function Controller($scope){ var dialogControl = $scope.$aoc('#editDialog').dialog; var person = {age:30,name:'Andrei',location:'Krakow'}; dialog.edit(person); }

HTML

<button ng-click="$aoc('#editDialog').dialog.edit(person)">Edit</button> <div aoc="dialog" id="#editDialog"></div>
